Netscape에서 처음 동적인 웹사이트를 제공
이후 브라우저 마다 동적 기능을 제공하는 언어가 난립함(표준화가 되지 않음)
2004년 Jesse James Garrett가 AJAX를 도입시킴
점점 웹개발에 개발자가 유입되면서 커뮤니티가 형성되고
jQuery, dojo mootools 라이브러리가 개발됨
다른 브라우저의 구현 사항을 신경쓰지 않아도 되도록
해당 라이브러리가 제공하는 api만 호출하면 어떤 브라우저에도 적용되도록 하는 라이브러리
그 중 jQuery가 많은 사랑을 받음
2008년 크롬 출시
jit 엔진이 포함되어 js 호출 속도가 매우 빠름
2008년 7월 브라우저 사들이 모여 표준화를 모의
2009년 ECMAScript 5, 2015년 ECMAScript 6가 나오게 됨
이후에도 매년 조금씩 발전된 버전이 나오고 있으나
대부분의 기틀은 5, 6에서 정립됨
모든 브라우저들이 ECMAScript 표준을 잘 따라가고 있어
jQuery 같은 라이브러리가 없더라도 모든 브라우저에서 잘 동작하는 웹사이트를 만들 수 있음

개발할 때는 ECMAScript 최신 버전을 사용하고,
사용자에게 제공할 때는 JavaScript transcompiler(BABEL)를 통해 ECMAScript 5, 6 버전으로 제공

동향
SPA(Single Page Application)가 뜨고 있음
하나의 페이지 안에서 데이터를 받아와서 필요한 부분만 업데이트 하는 것이 뜨고 있음
js로도 충분히 구현할 수 있지만 react, angular, vue 등을 통해 spa를 조금 더 쉽게 만들 수 있음

표준화와 v8 엔진을 기반으로 nodeJS 등장
백엔드에서 서비스 제공, 모바일 어플리케이션, 데스크톱 어플리케이션을 만들 수 있음

브라우저에서 동작할 수 있는 유일한 언어는 js였는데 요즘에는 웹어셈블리(WA)가 등장
다양한 언어를 이용해서 웹어플리케이션을 만드는 것이 가능해짐
